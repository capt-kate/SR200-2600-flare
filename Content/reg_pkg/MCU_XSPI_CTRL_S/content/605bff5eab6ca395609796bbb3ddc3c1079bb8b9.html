Description

Read command register. Configures the read command and data transfer mode to be used during automatic reads.

Field Descriptions

RESERVED_BIT

Reserved.

SKIP_CLOCKS

Skip clocks. This is the number of CK cycles which will be skipped, following the last dummy clock, before expecting the first bit of the return value.

DUMMY_CLOCKS

Dummy clocks. This is the number of CK cycles which will be sent follow the address before switching I/O direction.

Single_bit_enumeration

Contains a single bit enumeration indicating the type of read command that will be issued–whether a traditional 8’bit command, a 16’bit xSPI command, an eXecute–In–Place (XIP) command which would skip the command word, or a 48’bit profile–two command (xSPI only).

CM

Contains an enumeration holding the protocol mode of the command

CA

CA: True if the arguments use the same protocol mode as the command, false if they use the same protocol mode as the data. Hence, for example, if you wished to send a SPI command using QSPI arguments and QSPI data, something the xSPI and SFDP standards call 1S-4S-4S, you would set this flag false–the command doesn’t use the same I/O mode as the arguments. On the other hand, if you wanted to send a SPI command followed by SPI arguments, only to receive data in QSPI mode, what the standards call 1S-1S-4S, you might then set this flag to true since the command and arguments would use the same I/O mode.

DM

DM: The protocol mode to be used for the data portion of the interaction. It uses the same bit field as the command protocol mode, outlined in Tbl. 5.6, but this time applies to the data bits. One rule is that the data mode must be of an equal or higher speed mode than the command protocol mode.

CB

These bits contain the command byte(s) that will be sent to the device to begin a read command. They are what the xSPI specification calls RI, and so bits 15–8 must either equal bits 7–0 or be an inverted version of bits 7–0. Bits 15–8 will be sent on the rising edge of any xSPI command clock cycle, whereas bits 7–0 will be sent on the falling edge. In non–xSPI modes, bits 15–8 will be ignored. If Cb is set, then both bytes will be ignored.
