Field Descriptions

Busy

Busy. True if an override operation is pending that has either not (yet) been accepted, or for which the result hasn’t (yet) been returned. This bit will also be true while transitioning to or from override mode

Watchdog_error

Watchdog error: the xSPI device failed to toggle DS within the time frame specified in the watchdog timeout register

Valid_return_data

Valid return data: Not all override commands return responses. For those commands that do return responses, this bit will be set once the response has been received. It is cleared upon any override request of the device with the chip select active

Reserved

Reserved

Channel_selection

Channel selection. Controls which bit of CS# will be acti vated by a given override command. Leave at zero to activate o_xspi_csn[0]

Override_bit

Override bit. Once set, the controller is in command override mode. While in command override mode, requests on either the AXI slave port or from the DMA will result in SLVERR bus errors. While set, writes to this port will set the chip select, adjust the I/O protocol and possibly send or receive bits across the interface

Chip_select

Chip select bit. Writes to this register will adjust the chip select bit selected by bits 23–22, while setting all other chip selects high (inactive). Further, writes to the register with the chip select active (low), will also write the given data to the xSPI I/O port. The bit is active low, as are the actual outputs, so there’s no translation taking place here. This bit will be ignored unless the override bit is also set

Current_port_mode

Current port mode for potential port modes

xspi_mode

These bits will constitute the message sent when writing to this port, or alternatively the message received by the host. In all but xSPI mode, the upper 8-bits will be ignored in any transmit command. In xSPI mode, the upper 8-bits will be the first octet transmitted and the second octet received by the controller.
