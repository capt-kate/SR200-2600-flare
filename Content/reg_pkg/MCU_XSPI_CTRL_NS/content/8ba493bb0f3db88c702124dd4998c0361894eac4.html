TIMEOUT

Absolute Address:
Base Offset:
0xc
Size:
0x4

Description

The timeout register contains a watchdog timeout and further read configuration bits.

Contents

BitsIdentifierAccessResetDecodedName
[31] Watchdog_error_indicator r,rclr - - H
[30:20] watchdog_timeout_value r - - H
[19:16] Recycle__clock_cycles r - - H
[15:8] RESERVED_BITS r - - H

Field Descriptions

Watchdog_error_indicator

Watchdog error indicator. This bit will be set if the device ever fails to respond to an xSPI command by toggling the DS pins before the watchdog timer times out. This bit is sticky. It will get set if ever an error is detected, and only ever reset by writing a one to this bit. The bit is also mirrored in the configuration override register. In both registers, this error indicator may be cleared by writing a 1 to it.

watchdog_timeout_value

A watchdog timeout value. This is designed so as to be able to recover from a non-existent or mis-behaving xSPI device that doesn’t return data strobes as expected, and it’s only used in those cases.The watchdog timer is started in xSPI (4D or 8D) modes upon issuing a command to the device. If device does not toggle the DS input by the time the watchdog timer gets down to zero, the read request will return an error and set the watchdog error bit above.

Recycle__clock_cycles

Recycle clock cycles. This is the minimum number of slave CK clock cycle periods needed between commands where the CS# is deactivated.

RESERVED_BITS

reserved

XIP_mode

In order to support legacy XIP modes, where the read command is no longer issued, chips often require some number of “mode” bits that need to be issued following the read command else the memory will leave XIP mode. These mode bits can be set here.
