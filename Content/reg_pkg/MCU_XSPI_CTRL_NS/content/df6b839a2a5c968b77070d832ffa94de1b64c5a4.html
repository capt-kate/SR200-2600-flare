Field Descriptions

Reservedbitsfield

Reserved

IDly

IDly. Controls how far the incoming DS signal should be delayed,in units of one eigth of a DEV CLK cycle, in order to achieve the center of a data eye.

ReservedBits

Reserved

Odly

ODly. Controls the delay of the outgoing DS signal in units of one eighth of a DEV CLK cycle. (This field may be removed in future versions of this IP.) Set this field to zero.

Reserved

Reserved

CkDly

CkDly. This is the sub–clock delay control, represented in thirty-seconds of a DEV CLK period. The outgoing CK will be delayed by this amount in the PHY in order to generate a 90 degree phase shift. The field is only non–zero when the design has been built to interact with the ACS PHY.

INTN_bit

Mirrors the current state of the chip's INTN bit .This is different from bit [4], which is the state of the chips INTN saved that it may be masked by a subsequent user acknowledgment.

ECSN_bit

Mirrors the current state of the chip's ECSN bit .This is different from bit [3], which is the state of the chips INTN saved that it may be masked by a subsequent user acknowledgment.

Interrupt_Enable

IE. Interrupt Enable. If the INT# pin is connected, then this bit can be set in order to forward a device generated interrupt through the xSPI/PSRAM controller to the host. If this bit is clear, external interrupts will still be detected–just not forwarded out of the controller.

Device_Interrupt

DI. Device Interrupt. If the controller detects that the INT# pin is ever low, it will set this bit and, if enabled, forward the interrupt to the SOC. Write a one to this bit to clear it. Once cleared, however, the external INT# pin must return high before a second interrupt will be detected.

ECC_Error_detected

EC. ECC Error detected. This bit reflects the status of the ECS# pin. If ECS# ever goes low, this bit will be set. Write a one to this bit to clear it. Once cleared, however, the ECS# pin must return high before a second ECC error condition will be detected. (Note that the xSPI/PSRAM master controller has no ECC capability. This bit only reflects any ECC failure noted by the device.)

Write_Protect

Wp, Write protect. Controls the value in the outgoing WP# (o xspi wp n) pin, yet with the opposite sense. If set, the WP# pin will be clear. Further, if this write protect bit is set, then write requests made of the AXI slave port will result in SLVERR bus errors.

Reset_Input

RI, reset input. Returns the value of the RSTO# pin, but with the opposite sense. If high, this bit will indicate that the slave is still in its reset state.

Device_Reset

RS, device reset. Set this pin to one while in override to clear/activate the RESET# output pin. This bit may be cleared by either writing to it or by leaving override mode. This guarantees that any accesses to the device, while RESET# is active, will result in SLVERR bus errors.
