Absolute Address:

0x4

0x4

Description

Set control for each Hash operation.

Contents

BitsIdentifierAccessResetDecodedName
[31:4] - - - -
[3] reset rw 0x0 FW Reset Hash Control H
[2] hardware_padded_required rw 0x0 Hardware Padding Required H
[1:0] hash_op_mode rw 0x0 Hash Operation Mode For normal hash: H

Field Descriptions

reset - FW Reset Hash Control

FW set this bit to reset hash module all SM and control logic, This bit do not reset hash digest, key and context registers.

hardware_padded_required - Hardware Padding Required

1: Hardware padding required

0: Not need hardware padding

hash_op_mode - Hash Operation Mode For normal hash:

01: init, use standard constants as digest initial value

10: update, digest incoming message segment based on income message segment size, message size for this operation is module 64 bytes.

11: final, digest last portion of message, if hardware padded required bit is set, hash do hardware pad message. When final and hardware padding required set, incoming message size can be zero, and this means hash last padding block and generated digest.

For HMAC:

01: HMAC init, process key, generated: contexto="hash" (K0+opad) without padding, FW need save contexto for HMAC final use. contexti="hash" (K0+ipad) without padding, contexti will be in digest_context register and FW can read them as context for HMAC update's initial value.

10: HMAC update, digest incoming message

11: HMAC final: digest final portion of message with padding, that is contextm, load contextm to digest scheduler, load contexto as initial value, digest contextm, get final HMAC digest.
