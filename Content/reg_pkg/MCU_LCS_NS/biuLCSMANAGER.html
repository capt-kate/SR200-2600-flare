LCSMANAGER_LOCKSTS

32'h00000020

The $LCSMARRAY is allowed to be auto-loaded from OTP shadow copy after POR only once, after that $LCSMARRAY will be locked for any subsequent auto-load operation from OTP until next POR, the auto-load lock status can be read from this register bit LOCKSTS.allocked the dword (32-bit) in $LCSMARRAY can only be overwritten once by SW through APB interface, afterwards that dword will be locked for APB write, the lock status can only be cleared by POR/System reset. If LCS state falls into “corrupted” after auto-load, it will be locked immediately (no any override is allowed); for LCS shadow register, SW override can only be successful to advance LCS Manager state: LCSMDSTS.state, for OTP lock register, the bit value can only be modified from 0 to 1, from 1 to 0 is not allowed, unsuccessful SW override or any SW override after dword get locked will be ignored by LCS Manager. The dword will be locked (write prohibited) no matter SW override is successful or unsuccessful.
